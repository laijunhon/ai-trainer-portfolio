Obsidian Notes

从原始笔记提炼的行业知识

01 / 数据

AI 训练师不是“只写代码训练模型”

训练师更像给 AI 准备教材的人：整理文本、图片、视频、语音等学习材料，用数据质量影响模型生成效果。

02 / Prompt

PR、QA 与上下文是基础数据结构

一条 PR 数据由 prompt 和 response 组成；QA 对用于问答任务；上下文决定多轮对话里的角色、语气和约束。

03 / Agent

Agent = LLM + 工具调用

大模型负责思考，工具调用负责执行。Agent 的价值在于把“想法”变成“操作”，能规划任务、调用软件并完成复杂流程。

04 / Evaluation

评测不是打分，而是找改进路径

评测需要设计真实问题、边界问题和反常问题，记录模型答错、答偏、答不稳的原因，并形成可执行的优化建议。

05 / Annotation

标注方案要讲清“标什么、怎么判”

标注规则至少包含标注内容、query 理解、case 示例和备注口径，确保不同标注者对同一条数据有一致判断。

06 / API

API 调用三要素：地址、密钥、模型

Endpoint 决定请求去哪，API Key 决定权限，model 决定调用哪个能力。理解这三点，就能把 AI 接入工具链。
